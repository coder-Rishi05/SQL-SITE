SQL Joins

SQL Joins are used to combine rows from two or more tables based on a related column.

1. INNER JOIN

Returns only the rows with matching values in both tables.


            SELECT students.name, courses.course
            FROM students
            INNER JOIN courses ON students.id = courses.student_id;

Output:


            +--------+--------+
            | name   | course |
            +--------+--------+
            | Rishi  | Java   |
            | Ajay   | Python |
            +--------+--------+

2. LEFT JOIN

Returns all records from the left table and matched records from the right table. If no match, NULL values appear.


            SELECT students.name, courses.course
            FROM students
            LEFT JOIN courses ON students.id = courses.student_id;

Output:


            +--------+--------+
            | name   | course |
            +--------+--------+
            | Rishi  | Java   |
            | Ajay   | Python |
            | Priya  | NULL   |
            +--------+--------+

3. RIGHT JOIN

Returns all records from the right table and matched records from the left table.


            SELECT students.name, courses.course
            FROM students
            RIGHT JOIN courses ON students.id = courses.student_id;

Output:


            +--------+--------+
            | name   | course |
            +--------+--------+
            | Rishi  | Java   |
            | Ajay   | Python |
            | NULL   | PHP    |
            +--------+--------+

4. FULL OUTER JOIN

Returns all records when there is a match in either left or right table.


            SELECT students.name, courses.course
            FROM students
            FULL OUTER JOIN courses ON students.id = courses.student_id;

Output:


            +--------+--------+
            | name   | course |
            +--------+--------+
            | Rishi  | Java   |
            | Ajay   | Python |
            | Priya  | NULL   |
            | NULL   | PHP    |
            +--------+--------+

5. CROSS JOIN

Returns the Cartesian product of both tables (every combination).


            SELECT students.name, courses.course
            FROM students
            CROSS JOIN courses;

Output:


            +--------+--------+
            | name   | course |
            +--------+--------+
            | Rishi  | Java   |
            | Rishi  | Python |
            | Rishi  | PHP    |
            | Ajay   | Java   |
            | Ajay   | Python |
            | Ajay   | PHP    |
            | Priya  | Java   |
            | Priya  | Python |
            | Priya  | PHP    |
            +--------+--------+

6. SELF JOIN

A table joins with itself to compare rows within the same table.


            SELECT A.name AS Student1, B.name AS Student2
            FROM students A, students B
            WHERE A.id != B.id;

Output:


            +----------+----------+
            | Student1 | Student2 |
            +----------+----------+
            | Rishi    | Ajay     |
            | Rishi    | Priya    |
            | Ajay     | Rishi    |
            | Ajay     | Priya    |
            | Priya    | Rishi    |
            | Priya    | Ajay     |
            +----------+----------+

SQL Set Operators: UNION, INTERSECT, MINUS

Set operators in SQL are used to combine the results of two or more `SELECT` statements.

1. UNION

Combines results from two queries and removes duplicates.


            SELECT name FROM table1
            UNION
            SELECT name FROM table2;

Output:


            +--------+
            | name   |
            +--------+
            | Rishi  |
            | Ajay   |
            | Priya  |
            +--------+

2. INTERSECT

Returns only the rows that are common in both result sets.


            SELECT name FROM table1
            INTERSECT
            SELECT name FROM table2;

Output:


            +--------+
            | name   |
            +--------+
            | Ajay   |
            +--------+

3. MINUS

Returns rows from the first query that are not in the second.


            SELECT name FROM table1
            MINUS
            SELECT name FROM table2;

Output:


            +--------+
            | name   |
            +--------+
            | Rishi  |
            +--------+

Note: Some databases like MySQL do not support INTERSECT and MINUS directly. Use `JOIN` or `NOT IN` instead for such cases.

Introduction to Nested Queries in SQL

Nested Queries, also known as subqueries, are queries within another SQL query. They are used to perform operations that require multiple steps or to retrieve intermediate results.

Syntax:


              SELECT column_name
              FROM table_name
              WHERE column_name = (SELECT column_name FROM table_name WHERE condition);

Example Table: students


              +--------+--------+-------+
              | stId   | name   | marks |
              +--------+--------+-------+
              |  101   | Ajay   |   89  |
              |  102   | Riya   |   93  |
              |  103   | Mohan  |   76  |
              +--------+--------+-------+

Query:

Find the student(s) who scored the highest marks.


              SELECT name
              FROM students
              WHERE marks = (SELECT MAX(marks) FROM students);

Output:


              +--------+
              | name   |
              +--------+
              | Riya   |
              +--------+

Note: Subqueries can be used with SELECT, INSERT, UPDATE, DELETE, and can return single or multiple rows depending on usage.
